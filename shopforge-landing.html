What's included

Everything a real store needs.

No UI kit. No half-baked demo. Every feature is wired end-to-end — from database to browser.

🛒
Product Catalog
Multi-category listing with search, filter by category, featured highlights, and stock badges. Server-rendered for instant load.
🛍️
Persistent Cart
Zustand-powered cart that survives page reloads via localStorage. Quantity controls, line totals, and free-shipping threshold.
🔐
Auth System
NextAuth.js v5 with email/password credentials, bcrypt hashing, JWT sessions, and role-based access (USER / ADMIN).
💳
Checkout & Orders
Shipping address form, order placement via REST API, and a full order history page. Stripe-ready — just swap the payment step.
🛠️
Admin Dashboard
Revenue, orders, customers, and product stats at a glance. Add / edit products, update order statuses with a dropdown — all live.
🌱
Seed Script
One command loads 8 products across 5 categories, a demo admin, and a demo customer. Ready to demo in under a minute.

Ship it

From zero to live in 5 steps.

Deploy free on Vercel with a Neon PostgreSQL database. Both have generous free tiers.

01
Clone the repo
Fork or clone ShopForge from GitHub and install dependencies.
git clone https://github.com/you/shopforge && npm install
02
Create a database
Sign up for a free PostgreSQL instance at neon.tech or supabase.com. Copy the connection string.
03
Configure environment
Copy .env.example → .env.local and paste your DATABASE_URL and a generated NEXTAUTH_SECRET.
cp .env.example .env.local
04
Push schema & seed
Prisma pushes your schema to the database, then the seed script loads demo products, categories, and accounts.
npm run db:push && npm run db:seed
05
Deploy to Vercel
Push to GitHub, import the repo on vercel.com, add your env vars in the dashboard, and deploy. Done.
git push origin main # Vercel auto-deploys

Stop scaffolding.
Start shipping.
